## Measuring .NET performance Oliver Brennan

IT'S
SLOW

We should forget about small efficiencies, say about 97% of the time: premature optimization is the root of all evil. Yet we should not pass up our opportunities in that critical 3%.
The hope is that the progress in hardware will cure all software ills. However, a critical observer may observe that software manages to outgrow hardware in size and sluggishness.
### Why measure * optimise with confidence * have quantifiable performance goals * find regressions * it's easy to do!
### metrics * Wall-clock time * CPU * Time in user code * Time in GC * Memory * Allocation rate * Working set * JIT * I/O * Concurrency
### metric collection * profiling * performance counters * ETW * tracing
### tools * PerfView * dotMemory/dotTrace
